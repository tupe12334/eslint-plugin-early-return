# Detect unnecessary else after continue/break in loops

When a loopâ€™s if branch contains continue or break, the else block is redundant. The plugin should detect this and suggest removing the else.

**Bad:**

```javascript
for (const item of items) {
  if (item.skip) {
    continue
  } else {
    process(item)
    save(item)
  }
}
```

**Good:**

```javascript
for (const item of items) {
  if (item.skip) continue
  process(item)
  save(item)
}
```

Should handle continue, break, and labeled variants. Could be part of the prefer-early-continue rule or standalone.

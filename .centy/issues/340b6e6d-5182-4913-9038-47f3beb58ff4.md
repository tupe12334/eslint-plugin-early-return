# Detect unnecessary final else in if-else-if chains

When all preceding branches in an if-else-if chain exit (via return, throw, continue, or break), the final else is redundant and adds unnecessary nesting.

**Bad:**

```javascript
function handle(type) {
  if (!type) {
    throw new Error('missing')
  } else if (type === 'skip') {
    return null
  } else {
    doA()
    doB()
    return doC()
  }
}
```

**Good:**

```javascript
function handle(type) {
  if (!type) throw new Error('missing')
  if (type === 'skip') return null
  doA()
  doB()
  return doC()
}
```

Need to verify that all preceding if/else-if branches contain exit statements. Should handle mixed exit types (some return, some throw). Common in validation and dispatch code.

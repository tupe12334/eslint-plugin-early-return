# Detect unnecessary else after return/throw

When an if branch contains a return or throw, the else block is redundant and adds unnecessary nesting. The plugin should detect this and suggest removing the else.

**Bad:**

```javascript
function foo(x) {
  if (!x) {
    return null
  } else {
    doA()
    doB()
    return result
  }
}
```

**Good:**

```javascript
function foo(x) {
  if (!x) return null
  doA()
  doB()
  return result
}
```

Should handle both return and throw in the if branch. This is the inverse of the existing ‘if-else with simple exit in else’ pattern. One of the most common early-return anti-patterns in real codebases.
